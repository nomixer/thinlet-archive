Thinlet is a GUI toolkit, a single Java class, parses the hierarchy and properties of the GUI, handles user interaction, and calls business logic. Separates the graphic presentation (described in an XML file) and the application methods (written as Java code).

Its compressed size is 28KB, and it is LGPL licensed.

Thinlet runs with Java 1.1 (browsers' default JVM) to 1.4, Personal Java, Personal (Basis) Profile, and MIDP. Swing isn't required.
  
What's new
This fifth beta is released on October 28, 2002.
Parameters and more event handlers. Wrap textarea text. DOM like interface for the internal xml parser. The getSelectedIndex/Item(s) methods for list, table, and tree. Mousewheel works. JavaDoc generated APIdoc. Drawing modified for Zaurus and iPAQ. Horizontal gradient for vertical scrollbars. Togglebutton component. Panel border. Tooltip, dialog remains inside. FrameLauncher closing notification. Font bug fixed. The put/getProperty introducted (in the xml too) similar to Swing's put/getClientProperty. No NullPointerException for invalid image path. Combolist width. Bug fixed for resizeable applets, and small desktops.

What's next
Popup menu. Internal drag'n'drop for list, table, and tree. List item, tree node, table row, and combobox choice for event handling methods as parameter. Fix combobox bugs. More event types (e.g. for caret move, list double click). New thread model (e.g. repaint during long application tasks, entirely modal dialogs).
